# 架构实现说明

这是当前版本的架构摘要。更详细的方法定义和参数说明见 `工程详细说明.md`。

## 1. 总体架构

```text
UI / CLI
  |
  v
ebeam_backend.py
  |
  v
DuckDB SQL aggregation
  |
  v
pandas DataFrame
  |
  v
Matplotlib plot
```

## 2. 核心原则

- 大数据先在 DuckDB 中过滤、聚合、join。
- Python 端只接收聚合结果或抽样结果。
- `Value column` 是通用数值列，不限定为 `CD`。
- `ImageID` 是标准拍图 ID，旧字段 `ImageIDX` 作为别名兼容。
- Line loading 先在每个 `ImageID + line` 内求均值，再做 A/B 运算。
- Wafer heatmap 使用 X/Y 网格聚合，并支持固定色阶。

## 3. 主要功能和后端函数

| 功能 | 后端函数 |
|---|---|
| CD/value by radius | `query_radius_aggregate()` |
| Layer distribution | `query_layer_histogram()` |
| Line loading by radius | `query_line_loading()` |
| Custom scatter | `query_xy()` |
| Wafer value heatmap | `query_wafer_value_heatmap()` |
| Wafer loading heatmap | `query_wafer_loading_heatmap()` |

## 4. UI 动态控件

`ui_app.py` 中的 `update_control_visibility()` 根据 `Chart type` 控制左侧 UI 显示。

这样避免所有功能参数堆在一起。

## 5. Radius

虚拟字段：

```text
Radius = sqrt((WaferPosX - X0)^2 + (WaferPosY - Y0)^2)
```

300mm wafer 且坐标为 mm 时，常见设置：

```text
X0 = 0
Y0 = 0
Radius bin = 1, 2, 5, 10
```

## 6. Heatmap

Wafer heatmap 会：

1. 将坐标按 wafer center 平移。
2. 限制在 wafer 半径内。
3. 按 `Heatmap bin` 对 X/Y 分格。
4. 计算每格 value 或 loading 均值。
5. 使用 `Color min/max` 固定色阶。

不同 wafer 对比时必须使用相同 `Color min/max`。
