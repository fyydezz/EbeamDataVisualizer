# Ebeam Data Visualizer 工程详细说明

本文档面向维护者，说明当前工程架构、主要脚本、核心方法、数据粒度和验证方式。

## 1. 工程概览

工程目标：

- 支持 Ebeam 大表格数据可视化
- 支持亿级 CSV/Parquet 的聚合查询
- 支持 Windows UI 和命令行脚本
- 支持任意数值列作为 `Value column`
- 支持 ImageID 粒度均值后再做 line loading
- 支持 300mm wafer map heatmap 和固定色阶比较
- 支持 radius σ 趋势、FIN center loading、每根线独立颜色配置
- 支持分布图图例显示 `μ`、`σ`、`n`

## 2. 模块结构

| 文件 | 职责 |
|---|---|
| `ebeam_backend.py` | 数据读取、字段解析、SQL 聚合、核心查询 |
| `ui_app.py` | Tkinter UI、动态控件显示、Matplotlib 嵌入绘图 |
| `plot_cd_by_radius.py` | 命令行 radius trend |
| `plot_layer_cd_distribution.py` | 命令行 layer distribution |
| `plot_line_cd_loading.py` | 命令行 line loading by radius |
| `plot_radius_sigma.py` | 命令行 radius sigma trend |
| `plot_fin_center_loading.py` | 命令行 FIN center loading |
| `plot_wafer_heatmap.py` | 命令行 wafer value/loading heatmap |
| `generate_demo_data.py` | 生成测试数据 |
| `build_exe.ps1` | PyInstaller 打包 |

## 3. 数据模型

标准字段：

```text
DieIDX, DieIDY, WaferPosX, WaferPosY, ImageID, CD,
BendingAngle, LayerType, LayerNO, Moduleindex
```

兼容规则：

- `ImageID` 是标准字段
- 旧字段 `ImageIDX` 会被当作 `ImageID`
- 字段大小写和前后空格会尽量兼容

核心数据粒度：

- 原始 row：一条量测记录
- Image-level：同一个 `ImageID + line` 内多条记录求均值
- Radius bin：按半径分段聚合
- Wafer bin：按 X/Y 网格聚合

## 4. 大数据策略

CSV/TXT：

```python
read_csv_auto(..., header=true, ignore_errors=true, sample_size=200000)
```

Parquet：

```python
read_parquet(...)
```

Excel：

- 使用 pandas 读取
- 仅建议小数据测试

设计原则：

- 大表不直接全量读入 pandas
- 聚合、筛选、join 尽量在 DuckDB SQL 内完成
- Python 端只接收聚合结果、抽样结果或 heatmap grid 结果

## 5. ebeam_backend.py

### 常量

| 常量 | 说明 |
|---|---|
| `DEFAULT_COLUMNS` | 推荐字段 |
| `LAYER_COLUMNS` | layer/line 字段 |
| `JOIN_KEYS` | 当前为 `ImageID` |
| `RADIUS_COLUMN` | 虚拟列名 `Radius` |
| `COLUMN_ALIASES` | 字段别名，如 `ImageIDX` -> `ImageID` |

### DataSource

保存 DuckDB relation、connection 和文件路径。

### open_source(path, memory_limit, threads)

根据文件类型创建 DuckDB 可查询 relation。

支持：

- CSV/TXT
- Parquet
- 小型 Excel

### list_columns(path)

通过 DuckDB `DESCRIBE SELECT *` 获取字段名，不扫描全表。

### list_plot_columns(path)

如果存在 `WaferPosX/Y`，在字段列表中加入虚拟 `Radius`。

### resolve_column(columns, requested)

解析真实字段名。

能力：

- 完全匹配
- 忽略大小写
- 忽略前后空格
- 支持别名 `ImageIDX`

### require_columns(path, required, feature_name)

每个功能只检查自己需要的字段，避免因为缺少无关字段导致无法画图。

### radius_expr(x_col, y_col, center_x, center_y)

生成 SQL 半径公式：

```text
sqrt((WaferPosX - X0)^2 + (WaferPosY - Y0)^2)
```

### field_expr(field, columns, radius_center)

把 UI 选择的字段转换为 SQL 数值表达式。

- `Radius` 转成 `radius_expr`
- 普通字段转成 `try_cast(column AS DOUBLE)`

### layer_label_expr(columns, table_alias)

生成 series label：

```text
LayerType=M0 | LayerNO=1 | Module=0
```

缺字段时自动跳过。

### where_from_filters(filters, columns)

把 UI 或命令行筛选条件转换成 SQL WHERE。

支持单值和逗号多值。

### query_radius_aggregate(...)

用途：CD/value by radius。

主要参数：

- `value_col`
- `radius_bin`
- `filters`
- `group_series`
- `radius_center`

输出字段：

```text
radius_bin, series_label, value_mean, value_std,
value_min, value_max, n
```

实现：

1. 计算 radius
2. 按 `radius_bin` 生成 bin 中心
3. 按 bin 和 series 聚合

### query_radius_sigma(...)

用途：计算 value column 随 wafer radius 的离散程度。

主要参数：

- `value_col`
- `radius_bin`
- `filters`
- `group_series`
- `sigma_mode`
- `radius_center`

`sigma_mode`：

| 模式 | 实现 |
|---|---|
| `radius_bin` | 在每个 radius bin 内直接计算 `stddev_samp(value)` |
| `image` | 先在 `ImageID + series` 内计算 image-level σ，再按 radius bin 聚合 |

输出字段：

```text
radius_bin, series_label, sigma, n
```

设计原因：

- `radius_bin` 适合看整体半径区间内的总离散度
- `image` 适合一张图内有多个量测值时，先看每张 ImageID 内部的离散程度，再比较半径趋势

### query_xy(...)

用途：Custom X-Y scatter。

关键参数：

- `x_col`
- `y_col`
- `sample_rows`
- `use_sample`
- `aggregate_by_image`

`aggregate_by_image=True` 时，会在每个 `ImageID + series` 内计算 X/Y 均值，再抽样或全量返回。

### query_layer_histogram(...)

用途：layer 分布图。

返回：

- `hist`
- `stats`

实现：

1. 找出前 `max_layers` 个 series
2. 计算全局 min/max
3. 按 bins 聚合每个 series 的分布

`stats` 中包含：

```text
layer_label, n, mean, std, min_value, max_value
```

UI 和命令行脚本会把 `mean` 和 `std` 显示到 legend 中：

```text
LayerType=M0 | LayerNO=1 | Module=0 (μ=12.34, σ=0.056, n=12345)
```

### query_line_loading(...)

用途：Line A/B loading by radius。

实现：

1. 按 Line A filters 筛选 A
2. 按 Line B filters 筛选 B
3. A/B 分别在 `ImageID + line_label` 内求均值
4. 按 `ImageID` join
5. 计算 `A-B`、`A+B` 或 `A/B`
6. 按 radius bin 聚合

### query_fin_center_loading(...)

用途：计算三根 FIN 场景下的 center loading。

公式：

```text
((Line A + Line C) / 2) - Line B
```

实现：

1. 按 Line A/B/C filters 分别筛选三根线
2. A/B/C 各自在 `ImageID + line_label` 内求均值
3. 按 `ImageID` join 三根线
4. 计算 `((A + C) / 2) - B`
5. 按 radius bin 和 series 聚合

输出字段：

```text
radius_bin, series_label, loading_mean, loading_std,
loading_min, loading_max, n
```

### query_wafer_value_heatmap(...)

用途：wafer map 上显示某个 value column。

实现：

1. 坐标按 wafer center 平移
2. 限制在 wafer 半径内
3. 按 X/Y grid 聚合 value mean/min/max/count

输出：

```text
x_bin, y_bin, value_mean, value_min, value_max, n
```

### query_wafer_loading_heatmap(...)

用途：wafer map 上显示 Line A/B loading。

实现：

1. A/B 各自在 `ImageID + line_label` 内求均值
2. 按 `ImageID` join
3. 计算 loading
4. 按 X/Y grid 聚合

## 6. ui_app.py

### EbeamVisualizerApp

主 UI 类，继承 `tk.Tk`。

主要能力：

- 文件选择
- 字段读取
- 动态控件显示
- 后台线程查询
- 主线程绘图
- 保存 PNG

### 动态控件显示

核心方法：

```python
update_control_visibility()
```

作用：根据 `Chart type` 显示/隐藏左侧控件。

当前规则：

| Chart type | 显示重点 |
|---|---|
| `CD by wafer radius` | Value column, Radius bin, center, series, filters |
| `Radius sigma by wafer radius` | Value column, Radius bin, sigma mode, center, series, filters |
| `Layer CD distribution` | Value column, series, filters |
| `Line CD loading` | Value column, Radius bin, Line A/B, operation |
| `FIN center loading` | Value column, Radius bin, Line A/B/C |
| `Wafer value heatmap` | Value column, heatmap, color scale, filters |
| `Wafer loading heatmap` | Value column, heatmap, color scale, Line A/B |
| `Custom X-Y scatter` | X/Y axis, sample, scatter options |

### Plot style 和 series color

样式参数存放在 `StringVar` / `DoubleVar` 中，并通过 `_install_auto_style_traces()` 绑定自动刷新。

关键方法：

| 方法 | 作用 |
|---|---|
| `_install_auto_style_traces()` | 给颜色、线宽、点大小、坐标轴文字、色阶等样式项绑定 trace |
| `_schedule_style_redraw()` | 样式变更后延迟刷新，避免用户输入中途频繁重绘 |
| `_auto_apply_plot_style()` | 使用上一次查询结果重新绘图，不重新跑 DuckDB 查询 |
| `_series_color_map(labels)` | 解析 `Series colors` 输入 |
| `_series_style(index, first_color, label, color_map)` | 为每个 series 返回颜色和线型 |

`Series colors` 支持：

```text
# 按顺序指定
#d62728,#1f77b4,#2ca02c

# 按 label 片段指定
LayerNO=1:#d62728;LayerNO=2:#1f77b4
```

解析规则：

1. 先解析 `label片段:颜色`，只要片段能匹配完整 label 即可
2. 再把没有冒号的颜色按当前 label 顺序应用
3. 没指定的 series 继续使用默认调色板

### Distribution legend

分布图绘制时，`_draw()` 会从 `query_layer_histogram()` 返回的 `stats` 中取 `mean/std/n`，通过 `_distribution_legend_label()` 拼成图例 label。

示例：

```text
LayerNO=1 (μ=12.4, σ=0.08, n=35000)
```

### _plot_worker(path)

后台线程查询入口。

根据 chart type 调用：

| Chart type | Backend |
|---|---|
| CD by wafer radius | `query_radius_aggregate` |
| Radius sigma by wafer radius | `query_radius_sigma` |
| Layer CD distribution | `query_layer_histogram` |
| Line CD loading | `query_line_loading` |
| FIN center loading | `query_fin_center_loading` |
| Wafer value heatmap | `query_wafer_value_heatmap` |
| Wafer loading heatmap | `query_wafer_loading_heatmap` |
| Custom X-Y scatter | `query_xy` |

### _draw(kind, payload)

统一绘图入口。

绘图类型：

- radius line/std/minmax
- radius sigma line
- distribution curve
- line loading curve
- FIN center loading curve
- scatter
- wafer heatmap

### _draw_wafer_heatmap(...)

将 backend 返回的 grid 结果 pivot 成矩阵，再用 `imshow(interpolation="bilinear")` 绘制平滑 wafer map。

支持：

- wafer circle
- center crosshair
- fixed color min/max
- custom colormap
- 默认 `jet` colormap，便于显示红到蓝的 wafer 差异
