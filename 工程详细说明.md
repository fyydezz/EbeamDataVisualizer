# Ebeam Data Visualizer 工程详细说明

本文档面向 Python 初学者和后续维护者。目标不是只列函数名，而是说明这个项目为什么这样拆分、数据怎样从文件流到图表、每个脚本可以改哪里、出现问题时应该从哪里查。

## 1. 这个项目做什么

这个项目用于处理 Ebeam 量测数据和产线表格数据，并把常见分析图表做成 Windows UI 和命令行脚本。

它目前支持：

- CD 或其它数值列 by wafer radius
- 不同 layer/line 的分布图，并在图例中显示 `μ`、`σ`、`n`
- Line A/B loading by radius
- 三根 FIN 的 center loading：`((Line A + Line C) / 2) - Line B`
- Wafer value heatmap
- Wafer loading heatmap
- 自定义 X/Y 散点图
- 每根线单独设置颜色，方便 wafer-to-wafer 对比
- 大 CSV/Parquet 通过 DuckDB 聚合，避免一次性读入 pandas

## 2. 初学者先理解的整体流程

程序运行时大致分成 4 层：

```text
数据文件
  ↓
ebeam_backend.py：读取字段、筛选、聚合、计算
  ↓
ui_app.py 或 plot_xxx.py：拿到聚合结果
  ↓
Matplotlib：画图并保存/显示
```

如果你点击 UI 里的 `Plot`，流程是：

1. `ui_app.py` 读取当前 UI 参数，比如图表类型、Value column、Layer 筛选、bin size。
2. UI 在后台线程调用 `ebeam_backend.py` 中对应的查询函数。
3. backend 使用 DuckDB 直接查询 CSV/Parquet，尽量在 SQL 里完成筛选和聚合。
4. backend 返回一个 pandas DataFrame，通常已经是聚合后的较小结果。
5. UI 的 `_draw()` 根据结果画 Matplotlib 图。

如果你运行命令行脚本，比如：

```powershell
python plot_wafer_heatmap.py data.csv --mode value --value-col CD
```

流程类似，只是没有 Tkinter UI，脚本直接调用 backend，然后保存 PNG。

## 6. 数据字段和兼容规则

推荐字段：

```text
DieIDX
DieIDY
WaferPosX
WaferPosY
ImageID
CD
BendingAngle
LayerType
LayerNO
Moduleindex
```

字段说明：

| 字段 | 说明 |
|---|---|
| `WaferPosX`, `WaferPosY` | wafer 坐标，通常是 mm，300mm wafer 常见范围约 `-150 ~ +150` |
| `ImageID` | 拍图 ID，很多计算会先按 ImageID 聚合 |
| `CD` | 默认量测值 |
| `BendingAngle` | 也可以作为 Value column |
| `LayerType`, `LayerNO`, `Moduleindex` | 用于区分不同 line/layer |

兼容规则在 `ebeam_backend.py` 里：

- `ImageIDX` 会被当作 `ImageID`
- 字段名会尽量忽略大小写和前后空格
- 某个功能只检查自己需要的列，不要求所有标准字段都存在

例如，只画 `CD vs WaferPosX` 散点图时，不一定需要 `LayerType`。

## 7. 数据粒度

理解“粒度”很重要。

| 粒度 | 含义 |
|---|---|
| 原始 row | CSV 里的一行量测记录 |
| Image-level | 同一个 `ImageID + line` 内的多条记录先求均值 |
| Radius bin | 按 wafer 半径分段，例如 0-2mm、2-4mm |
| Wafer bin | 按 X/Y 网格分格，例如 5mm x 5mm |

为什么 loading 要先算 Image-level？

因为 Ebeam 是先拍图，再在同一张图里量多个 CD。如果直接拿所有 row 做 A/B join，会产生错误的多对多组合。现在的逻辑是：

1. 每个 `ImageID + Line A` 先求均值。
2. 每个 `ImageID + Line B` 先求均值。
3. 按 `ImageID` 配对。
4. 再算 A-B、A/B 或 FIN 公式。

## 8. bin 计算为什么用整数 ID

这是当前工程里一个重要修复点。

以前有些地方直接用浮点中心坐标做分组，例如：

```text
floor(radius / bin) * bin + bin / 2
```

这在数学上没有问题，但计算机里的浮点数可能出现极小误差。比如本来应该是 `12.5`，内部可能变成 `12.49999999997` 或 `12.50000000003`。如果后续拿这些浮点数做对齐，可能就对不上，表现为：

- 小 bin 时图像空白
- heatmap 矩阵很多位置是 NaN
- bin 调大到 30/40 才能看见图

现在的做法是：

1. 先算整数 bin id。
2. SQL 里按整数 id 分组。
3. 最后只在输出给图表时，把 id 换算成显示坐标。

例如 radius：

```text
radius_bin_id = floor(radius / radius_bin)
radius_bin = radius_bin_id * radius_bin + radius_bin / 2
```

例如 wafer heatmap：

```text
x_bin_id = floor((x + wafer_radius) / bin_size)
y_bin_id = floor((y + wafer_radius) / bin_size)
x_bin = x_bin_id * bin_size - wafer_radius + bin_size / 2
y_bin = y_bin_id * bin_size - wafer_radius + bin_size / 2
```

这样 `GROUP BY` 和矩阵填充都基于整数 id，避免浮点对齐问题。

## 9. ebeam_backend.py 详解

`ebeam_backend.py` 是整个项目最核心的文件。它不负责画图，只负责把大数据文件变成适合画图的小表。

### 9.1 常量

| 名称 | 作用 |
|---|---|
| `DEFAULT_COLUMNS` | 推荐字段列表 |
| `LAYER_COLUMNS` | `LayerType`, `LayerNO`, `Moduleindex` |
| `JOIN_KEYS` | 当前主要是 `ImageID` |
| `RADIUS_COLUMN` | UI 中虚拟列名 `Radius` |
| `COLUMN_ALIASES` | 字段别名，例如 `ImageIDX` -> `ImageID` |

### 9.2 DataSource

`DataSource` 是一个小的 dataclass，用来保存：

- DuckDB connection
- DuckDB relation SQL
- 文件路径

初学者可以把它理解成“打开的数据源句柄”。

### 9.3 open_source(path, ...)

作用：根据文件类型创建 DuckDB 可查询对象。

支持：

- `.csv`
- `.txt`
- `.parquet`
- `.pq`
- 小型 `.xlsx`

大 CSV 不会先完整读入 pandas，而是交给 DuckDB 扫描。

### 9.4 list_columns(path)

作用：读取文件字段名。

UI 点击 `Load Columns` 时会调用它。

### 9.5 resolve_column(columns, requested)

作用：把用户选择的字段名解析成真实字段名。

例如：

```text
requested = ImageID
真实文件里叫 ImageIDX
```

它会通过 `COLUMN_ALIASES` 找到真实列。

### 9.6 require_columns(path, required, feature_name)

作用：检查某个功能需要的列是否存在。

例如 heatmap 必须有：

```text
WaferPosX
WaferPosY
Value column
```

如果缺列，会抛出清晰的错误。

### 9.7 radius_expr(...)

作用：生成 SQL 半径公式。

公式：

```text
sqrt((WaferPosX - X0)^2 + (WaferPosY - Y0)^2)
```

`X0/Y0` 来自 UI 的 `Wafer center for Radius`。

### 9.8 where_from_filters(...)

作用：把 UI 筛选条件转成 SQL WHERE。

例如 UI 输入：

```text
LayerType = M0,M1
LayerNO = 1,2
```

会变成类似：

```sql
WHERE LayerType IN ('M0', 'M1')
  AND LayerNO IN ('1', '2')
```

### 9.9 query_radius_aggregate(...)

用途：画 `CD by wafer radius` 或其它 value column by radius。

输入重点：

- `path`
- `value_col`
- `radius_bin`
- `filters`
- `group_series`
- `radius_center`

输出字段：

```text
radius_bin
series_label
value_mean
value_std
value_min
value_max
n
```

实现逻辑：

1. 计算 radius。
2. 计算整数 `radius_bin_id`。
3. 按 `radius_bin_id + series_label` 聚合。
4. 输出平均值、标准差、最小值、最大值、数量。

### 9.10 query_radius_sigma(...)

用途：画 `Radius sigma by wafer radius`。

两种模式：

| `sigma_mode` | 含义 |
|---|---|
| `radius_bin` | 直接在每个 radius bin 内计算所有点的 σ |
| `image` | 先在每个 `ImageID + series` 内算 σ，再按 radius 聚合 |

输出字段：

```text
radius_bin
series_label
sigma
n
```

### 9.11 query_xy(...)

用途：自定义 X/Y 散点图。

重要参数：

- `x_col`
- `y_col`
- `sample_rows`
- `use_sample`
- `aggregate_by_image`

如果 `aggregate_by_image=True`，程序会先在每个 `ImageID + series` 内求 X/Y 均值，减少同一张图内大量重复点。

### 9.12 query_layer_histogram(...)

用途：画不同 layer/line 的分布图。

返回两个 DataFrame：

| 返回值 | 含义 |
|---|---|
| `hist` | 每个 bin 的数量，用于画分布曲线 |
| `stats` | 每个 series 的统计值，用于 legend |

`stats` 包含：

```text
layer_label
n
mean
std
min_value
max_value
```

UI 会把它显示成：

```text
LayerNO=1 (μ=12.4, σ=0.08, n=35000)
```

### 9.13 query_line_loading(...)

用途：Line A/B loading by radius。

支持：

| operation | 公式 |
|---|---|
| `subtract` | A - B |
| `add` | A + B |
| `ratio` | A / B |

实现：

1. 按 A 筛选条件筛出 A。
2. 按 B 筛选条件筛出 B。
3. A/B 各自在 `ImageID + line_label` 内求均值。
4. 按 `ImageID` join。
5. 计算 loading。
6. 用整数 `radius_bin_id` 聚合。

### 9.14 query_fin_center_loading(...)

用途：三根 FIN 的 center loading。

公式：

```text
((Line A + Line C) / 2) - Line B
```

典型含义：

- Line A：上 FIN，常见 `LayerNO=1`
- Line B：中 FIN，常见 `LayerNO=2`
- Line C：下 FIN，常见 `LayerNO=3`

### 9.15 query_wafer_value_heatmap(...)

用途：wafer map 上显示某个 value column。

输出字段：

```text
x_bin
y_bin
value_mean
value_min
value_max
n
```

实现：

1. 坐标减去 wafer center。
2. 只保留 wafer 圆内数据。
3. 计算整数 `x_bin_id/y_bin_id`。
4. 按整数 bin id 聚合。
5. 输出用于显示的 `x_bin/y_bin`。

### 9.16 query_wafer_loading_heatmap(...)

用途：wafer map 上显示 A/B loading。

实现类似 `query_line_loading()`，但最后按 X/Y wafer bin 聚合，而不是按 radius bin 聚合。

## 10. ui_app.py 详解

`ui_app.py` 负责图形界面。它包含一个主类：

```python
class EbeamVisualizerApp(tk.Tk):
```

初学者可以理解成：整个窗口就是这个类。

### 10.1 __init__()

`__init__()` 做几件事：

1. 设置窗口标题、大小。
2. 创建很多 `StringVar`、`DoubleVar`、`BooleanVar`。
3. 调用 `_build_ui()` 创建控件。
4. 调用 `update_control_visibility()` 根据图表类型隐藏/显示控件。
5. 启动 `_poll_results()` 定时检查后台线程结果。

### 10.2 Tkinter 变量

UI 里的输入框通常绑定到变量，例如：

```python
self.value_col = tk.StringVar(value="CD")
self.radius_bin = tk.DoubleVar(value=5.0)
self.series_mode = tk.StringVar(value="Overlay selected series")
```

当用户在界面上修改输入框时，这些变量的值也会改变。

### 10.3 _build_ui()

作用：创建左侧控制面板和右侧 Matplotlib 图表区域。

左侧是：

- Browse
- Load Columns
- Chart type
- Value column
- Radius bin
- Heatmap 参数
- Layer 筛选
- Line A/B/C 筛选
- Plot style
- Plot / Save PNG

右侧是 Matplotlib canvas。

### 10.4 update_control_visibility()

作用：根据图表类型，隐藏不相关控件。

例如选择 `Wafer value heatmap` 时，不需要显示 Line A/B/C；选择 `FIN center loading` 时，需要显示 Line A/B/C。

这让 UI 不那么拥挤。

### 10.5 start_plot()

点击 `Plot` 时调用。

它不会直接在主线程查询数据，而是启动后台线程：

```python
threading.Thread(target=self._plot_worker, args=(path,), daemon=True)
```

这样大数据查询时 UI 不会完全卡死。

### 10.6 _plot_worker(path)

后台线程入口。

它根据 `chart_type` 判断应该调用哪个 backend 函数。

| Chart type | 调用函数 |
|---|---|
| `CD by wafer radius` | `query_radius_aggregate()` |
| `Radius sigma by wafer radius` | `query_radius_sigma()` |
| `Layer CD distribution` | `query_layer_histogram()` |
| `Line CD loading` | `query_line_loading()` |
| `FIN center loading` | `query_fin_center_loading()` |
| `Wafer value heatmap` | `query_wafer_value_heatmap()` |
| `Wafer loading heatmap` | `query_wafer_loading_heatmap()` |
| `Custom X-Y scatter` | `query_xy()` |

查询完成后，它把结果放进 queue：

```python
self.result_queue.put(("radius", df))
```

### 10.7 _poll_results()

主线程每隔一段时间检查 queue。

如果发现后台线程放入了结果，就调用 `_draw(kind, payload)` 画图。

### 10.8 _draw(kind, payload)

统一绘图入口。

它根据 `kind` 判断画哪种图：

- `radius`
- `radius_sigma`
- `layer_dist`
- `line_loading`
- `fin_center_loading`
- `scatter`
- `wafer_value_heatmap`
- `wafer_loading_heatmap`

### 10.9 _draw_wafer_heatmap()

作用：把 backend 返回的 heatmap 聚合表变成二维矩阵，再用 Matplotlib 画出来。

重要步骤：

1. `_heatmap_matrix()` 按整数 index 填矩阵。
2. `_fill_heatmap_gaps()` 对 wafer 内部空白格做显示层插值。
3. `imshow(interpolation="bilinear")` 画平滑 wafer map。
4. 画 wafer 圆边界和中心线。

注意：插值只用于显示，不会改变 backend 聚合出来的原始统计值。

### 10.10 Plot style

Plot style 相关方法：

| 方法 | 作用 |
|---|---|
| `_install_auto_style_traces()` | 绑定样式输入框的自动刷新 |
| `_schedule_style_redraw()` | 延迟刷新，避免输入一个字符就重画 |
| `_auto_apply_plot_style()` | 用上一次查询结果重画 |
| `_series_color_map()` | 解析 `Series colors` |
| `_series_style()` | 返回每根线的颜色和线型 |

`Series colors` 支持：

```text
# 按顺序
#d62728,#1f77b4,#2ca02c

# 按 label 片段
LayerNO=1:#d62728;LayerNO=2:#1f77b4
```

## 12. 常见修改需求应该改哪里

| 需求 | 优先修改文件 |
|---|---|
| 新增一种图表 | `ui_app.py` + `ebeam_backend.py`，必要时新增 `plot_xxx.py` |
| 新增筛选字段 | `ebeam_backend.py` 的 filter/label 相关逻辑，`ui_app.py` 的控件 |
| 改 UI 布局 | `ui_app.py` 的 `_build_ui()` 和 `update_control_visibility()` |
| 改 heatmap 样式 | `ui_app.py` 的 `_draw_wafer_heatmap()`，以及 `plot_wafer_heatmap.py` |
| 改 loading 公式 | `ebeam_backend.py` 的 `query_line_loading()` 或 `query_fin_center_loading()` |
| 改打包方式 | `build_exe.ps1` 和 `.spec` 文件 |
| 改文档 | `脚本使用说明.md`、`UI使用介绍.md`、`工程详细说明.md` |

## 13. 调试方法

### 13.1 先确认字段

如果报缺字段，先确认 CSV 真实列名。

可以用 Python：

```powershell
python -c "import pandas as pd; print(pd.read_csv('data.csv', nrows=0).columns.tolist())"
```

### 13.2 单独跑 backend 查询

如果 UI 画不出来，可以先单独跑查询：

```powershell
python - <<'PY'
from ebeam_backend import query_radius_aggregate
df = query_radius_aggregate('data.csv', value_col='CD', radius_bin=2)
print(df.head())
print(df.shape)
PY
```

### 13.3 检查是否真的查到数据

常见原因：

- Layer 筛选值不存在
- Line A/B/C 不在同一个 ImageID 中
- Value column 不能转成数字
- WaferPosX/Y 单位和 bin size 不匹配
- X/Y range 手动设置后没有清空

```powershell
python -m py_compile ebeam_backend.py ui_app.py
```

```powershell
python - <<'PY'
import ast
```
